既存ナレッジやテンプレートを下敷きに、**文書の草案を生成**するユースケースです。
回答 Q&A と違い、より多くの文脈を投入し、体裁・構成を整える力が求められます。

## 要件

- 社内テンプレート・用語・トーンに沿わせる
- 参照した根拠を脚注やコメントで残せる
- 長文生成のためコンテキスト量とコストの設計が重要

## 典型フロー

```mermaid
flowchart LR
    REQ[作成依頼 + 雛形] --> RET[関連文書を検索]
    RET --> CTX[テンプレ + 根拠で文脈構築]
    CTX --> GEN[セクション単位で生成]
    GEN --> ASM[組み立て / 整形]
    ASM --> DRAFT[ドラフト]
```

## 勘所

- **テンプレート駆動** にすると体裁が安定する
- セクション分割生成でコンテキスト超過とコストを抑える → [最適化](/ai-tech-notes/cost-roi/optimization/)
- 参照元を保持し、後段の[レビュー](/ai-tech-notes/use-cases/review-assist/)へ引き継ぐ
