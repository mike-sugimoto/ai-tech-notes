作成物を、**明文化された観点・基準に沿ってレビュー**するユースケースです。
基準（チェックリスト・ガイドライン）をどう与えるかが品質を決めます。

## 要件

- レビュー観点を **基準ドキュメント** として明示的に渡す
- 指摘には根拠（基準のどの項目に違反するか）を添える
- 誤検知（過剰指摘）と見逃しのバランスを取る

## 典型フロー

```mermaid
flowchart LR
    DOC[レビュー対象] --> STD[基準/チェックリスト取得]
    STD --> EVAL[観点ごとに評価]
    EVAL --> FIND[指摘 + 根拠]
    FIND --> SUM[サマリ + 重大度]
```

## 勘所

- 基準そのものをナレッジ化し RAG で参照させる
- 重大度でフィルタし、ノイズを減らす
- コードレビューでは [GitHub](/ai-tech-notes/data-sources/github/) 連携が中心
